LISTA DE REQUISITOS (RF e RNF) 

Funcionais:
O sistema deve permitir ao usuário realizar depósitos e saques. [RF001]
O sistema deve permitir opções de prazo mínimos para saque. [RF002]
Caso o usuário saque antes do prazo cobrar uma taxa administrativa de 0,5%. [RF003]
O sistema deve permitir gerar relatórios de rendimentos para que o usuário (investidor) acompanhe os seus ganhos. [RF004]
O sistema deve permitir que o usuários (gestor de investimento) faça investimentos com o dinheiro depositado pelos investidores. [RF005]
O sistema deve permitir que o usuário (gestor de investimentos) tenha acesso ao dashboard de ações.  [RF006]
O sistema deve permitir que o usuário (gestor de investimentos) acompanhe a progressão de ganhos da sua carteira de investimentos. [RF007]
O sistema deve permitir que o usuário (administrador) realize o cadastro de novos usuários (administrador, gestor e investidor). [RF008]
O sistema deve permitir ao administrador definir um valor limite para compra de investimentos (gestor), sendo sempre menor que o valor de dinheiro em caixa e definir a meta de rendimento mensal para cada gestor. [RF009]
O sistema deve permitir que o usuário (administrador) acompanhe a progressão de ganhos da empresa (fluxo de caixa), por gestor e por investidor. [RF010]
O sistema deve conter vários filtros para visualização dos dados como por poeríodo, por investimentos e etc... [RF011]
O sistema deve ter uma porcentagem máxima para o fundo de investimento, e o restante deve ser destinado ao fundo de saques. [RF012]
Se não houver nenhum cadastro de usuários (investidor, gestor ou administrador) no banco de dados o sistema deve permitir a criação de uma usuário administrador. [RF013]

Não Funcionais:
Botões de compra e venda de fácil acesso. [RNF001]
O sistema deve ter disponibilidade em 99,9% do tempo. [RNF002]
O sistema deve ter desenvolvimento em linguagem de programa web PHP 7. [RNF003]
O sistema deve rodar em um hardware com no minimo: processador i3, RAM de 4GB e memória de 500GB. [RNF004]
Os relátorio não podem exceder o prazo de 5s para serem gerados. [RNF005]
Deve ser possível realizar a atualização do sistema sem precisar pará-lo. [RNF006]
